# LooFinder Constitution

## Core Principles

### I. Simplicity-First
Every decision favors the simplest approach. Avoid complexity until it is clearly justified by concrete needs.

### II. Technology Stack
- **Runtime**: TypeScript and Node.js
- **Terminal UI**: Ink (React for CLIs)
- **Dependencies**: Minimal — prefer standard library and built-in platform features over third-party packages

### III. Code Quality
- No premature abstraction
- All code should be readable without comments
- Prefer composition over inheritance

### IV. Error Handling
Error handling must be explicit. Never silent failures. Errors must be surfaced and handled deliberately.

## Governance

This constitution supersedes all other practices. Amendments require documentation and approval.

**Version**: 1.0.0 | **Ratified**: 2025-03-10
